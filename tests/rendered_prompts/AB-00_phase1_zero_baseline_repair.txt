Build commands (e.g. `cargo build`) run without sudo. A `cargo: Permission denied` / exit 127 means a home-dir toolchain installer was run with `sudo` (rustup landed in `/root/.cargo`, unreachable by this user) — reinstall it WITHOUT sudo so it lands in the user's home, or install the toolchain via `sudo apt-get install` instead. Do not chmod `/root`.

For a Gradle toolchain error ("Cannot find a Java installation ... matching: {languageVersion=N}") that the pre-installed JDK does not satisfy, prefer letting Gradle's toolchain auto-download fetch that JDK, or disable toolchain enforcement and point the build at the installed JDK (`-Porg.gradle.java.installations.paths=$JAVA_HOME`), or relax/remove the toolchain `languageVersion` pin; apt-install a versioned JDK only if Ubuntu 24.04 carries that major.

Do NOT reformat, reorder, rename, or "clean up" steps that already work; gratuitous rewrites regress working builds. You MAY change an earlier step if the diagnosis shows it is the root cause, but never touch a working step without a reason from the failure evidence.
